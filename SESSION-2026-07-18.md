## Why opendata.host, not JustizOnline HVD?

The BMJ JustizOnline HVD SOAP API (`/jop/api/at.gv.justiz.fbw/ws`) is the
authoritative source for fresh registry data, but it requires ID Austria +
IWG (Identitätsverbund Österreich) credentials and a signed declaration
processed manually by BMJ — typical lead time 4–12 weeks. opendata.host is
the only paid-drop-in alternative that exposes the same FN-indexed
lookups over HTTP REST with a 1-look-up-per-call key. Its data is
documented to be a snapshot (May 2026 verified stale for some entities)
and not authoritative for "still active" verdicts.

For strict-verified (operating + non-insolvent) classification we need at
least one insolvency source. We do not have Ediktsdatei IWG credentials
either, so the current scout is `REGISTRY_OPEN_*` only.
